# Notes: blog post on working well with AI

Working notes for an upcoming "here are my thoughts" post. Direction has been revised; see the
companion file `working-with-ai-post-plan.md` for the locked outline.

## Raw themes (original list, numbered for reference)
1. Work that "just needs to be done" vs work that needs to be done *well*
2. Iterate (enrich context) vs start over (clear context)
3. Deterministic vs non-deterministic outcome: let AI make the thing directly, or insert an
   abstraction layer (AI makes the *tool* that makes the thing)
4. Does AI make humans lazier? Tendency probably yes, depends how you use it
5. Counter-example to laziness: building a functional-programming course from scratch.
   How do I *extract* what I need from the vast corpus?
6. Loop closing: the agent doesn't live with the consequences of its actions, unless you make it so
7. What requires AI vs what requires a human? "Glad it just exists" vs "I want a human interaction
   here." Ties back to just-done vs well-done.
8. What NOT to use AI for: repetitive tasks. But AI can build the automation for them! Determinism again.
9. Converging toward the optimal solution through iteration/loops (humans do this too)
10. Huge variation in knowledge quality in the corpus. "Monkey see, monkey do." Crap codebase ->
    agent adds crap, because it's pattern-matching on the crap it sees.
11. Sycophancy: gives the user a feeling of expertise quickly, when really they're taking shortcuts
    and lying to themselves. (There's a name for this.) Aware of the irony of writing this with AI.

## Name for the "false expertise" effect (theme 11)
- "Illusion of explanatory depth" -- feeling you understand a thing more deeply than you do.
  Best fit. Name-check it lightly, don't lecture.
- Cousins: Dunning-Kruger (skill self-assessment), cognitive offloading / automation complacency
  (the mechanism, not the feeling).
